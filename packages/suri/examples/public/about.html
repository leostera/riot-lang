📖 About Static Files Middleware

High-performance static file serving for OCaml

What is it?

The Static Files Middleware is a production-ready solution for serving static assets (HTML, CSS, JavaScript, images, etc.) in Riot/Suri web applications.

It provides efficient file delivery with built-in security, caching, and developer-friendly features like optional directory browsing.

Security Features

  • Path Traversal Prevention - All paths are canonicalized and validated
  • Dotfile Blocking - Prevents access to .env, .git, and other sensitive files
  • Symlink Control - Configure whether to follow or deny symbolic links
  • File Type Validation - Only serves regular files, not special devices

Performance Optimizations

  • ETag Generation - Unique identifiers based on file size and modification time
  • Last-Modified Headers - Enables browser cache validation
  • 304 Not Modified - Reduces bandwidth by serving cached content
  • Cache-Control - Configurable caching strategies

Configuration

let config = Static.{
  show_directory = true;
  index_files = ["index.html"; "index.htm"];
  dotfiles = `Deny;
  symlinks = `Follow;
  headers = [
    ("x-content-type-options", "nosniff");
  ];
  cache_control = Some "public, max-age=3600";
}

let app = Middleware.[
  static ~at:"/public" ~config (Path.v "./public") ();
  router routes;
]
