The `Toolchain` tab is the primary Riot tab because it is where most users should spend most of their
time.

It answers the questions that matter during adoption:

- how do I install Riot?
- how do I create a workspace?
- what is the normal build/run/check loop?
- what does Riot bring with it besides the CLI?
- where does the actor runtime fit?

If Riot is going to convert a curious user into a serious user, this tab has to do that work.

## The adoption path

If you are still evaluating Riot, read these pages in order:

1. [Installation](/toolchain/installation)
2. [Quick Start](/toolchain/quickstart)
3. [Editor Setup](/toolchain/editor-setup)
4. [riot build](/toolchain/build)
5. [riot run](/toolchain/riot-run)

That is the shortest path from `what is this?` to `I can actually use this`.

## The daily loop

Once a workspace exists, the center of gravity usually becomes:

- [riot build](/toolchain/build)
- [riot check](/toolchain/check)
- [riot fix](/toolchain/fix)
- [riot fmt](/toolchain/fmt)
- [riot run](/toolchain/riot-run)

Those commands are not five unrelated features. Together they define the normal edit, validate, and
execute rhythm of a Riot workspace.

## What to expect from this section

These pages are not intended to be terse reference blurbs. They should:

- teach the mental model
- show the common workflow
- explain what a page is good for
- point you to the next layer down when exact API details matter

If a page here does not make you more effective after reading it, it is not doing its job.
