---
title: "riot fix"
description: "Apply safe automatic corrections so routine issues stop consuming human attention."
---

`riot fix` is Riot’s "let the tool do the mechanical part" command.

It exists for issues where the right repair is already known and can be applied safely.

## Why this is valuable

Without a fix path, teams waste time on work like:

- repetitive rewrites
- low-level lint cleanups
- changes that are too boring to justify human attention but too common to ignore

`riot fix` moves those repairs into the toolchain where they belong.

## When to use it

The most normal pattern is:

```sh
riot check
riot fix
riot check
```

That sequence makes sense because:

- `riot check` tells you what is wrong
- `riot fix` applies what can be repaired automatically
- the second `riot check` confirms the new state

## What it should not be

`riot fix` should not be an excuse for unclear code ownership or blind rewrites.

It is useful when:

- the repair is mechanical
- the transformation is stable
- the result is easier to trust than a hand edit

If those conditions do not hold, the repair probably should stay manual.
