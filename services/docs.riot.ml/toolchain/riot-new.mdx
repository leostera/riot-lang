---
title: "riot new"
description: "Scaffold new package or entrypoint surface inside an existing Riot workspace."
---

`riot new` is the command you reach for once the workspace already exists and you want to add more
surface area without reconstructing the layout by hand.

It is the "grow the workspace" counterpart to `riot init`.

## When it is useful

Use `riot new` when you want to add:

- another package
- another binary entrypoint
- another starter layout that should still follow Riot conventions

The value is not only speed. The value is consistency:

- naming stays predictable
- layout stays recognizable
- the new surface starts inside the normal Riot flow instead of as a special case

## Why this matters

As workspaces grow, manual scaffolding becomes a source of drift. Two packages solve the same problem
with two slightly different structures, and eventually the docs and tooling have to explain both.

`riot new` is one way to keep that drift down.

## After creating something new

The next checks are the same as always:

```sh
riot build
riot run <name>
riot test
```

The best scaffold is one that disappears immediately into the normal workflow.
