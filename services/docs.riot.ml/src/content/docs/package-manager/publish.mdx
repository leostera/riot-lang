---

`riot publish` is where local package work becomes ecosystem work.

## What a good publish flow does

- validates the package surface before upload
- treats version and metadata as part of the package contract
- aligns docs, artifacts, and registry identity

## What to check before publishing

- the package still builds from a clean state
- the tests you care about pass
- public docs are useful enough to land in front of strangers
- dependency and version state are where you want them

Publishing should feel like a deliberate release event, not a blind upload.
