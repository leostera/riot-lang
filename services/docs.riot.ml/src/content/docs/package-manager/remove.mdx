---
title: "riot remove"
description: "Remove a dependency cleanly from the package manifest and package graph."
---

`riot remove` is the reverse of `riot add`.

Use it when a dependency should stop being part of the package contract instead of merely going unused
locally.

## Why a command matters here

Removing a dependency should keep the workspace coherent:

- manifests stop mentioning it
- lockfile state can be updated accordingly
- future builds do not carry accidental baggage

The command exists so dependency cleanup is as first-class as dependency growth.
