Riot’s package manager is not a separate product bolted onto the toolchain. It is the part that turns
package workflows into something repeatable and publishable.

<div class="home-grid home-grid--cards">
  <a class="home-card" href="/package-manager/add/">
    <strong>riot add</strong>
    <span>Add dependencies and let Riot update manifests, solve versions, and refresh the lock state.</span>
  </a>
  <a class="home-card" href="/package-manager/install/">
    <strong>riot install</strong>
    <span>Materialize runnable binaries and tools in a way that still respects Riot’s package model.</span>
  </a>
  <a class="home-card" href="/package-manager/publish/">
    <strong>riot publish</strong>
    <span>Take a package from local development to a publishable artifact without hand-assembling the release flow.</span>
  </a>
  <a class="home-card" href="/package-manager/doc/">
    <strong>riot doc</strong>
    <span>Generate package docs from public interfaces and keep them aligned with the package graph.</span>
  </a>
</div>

## What this tab covers

- adding and removing dependencies
- updating and searching packages
- installing runnable binaries
- publishing to the registry
- generating package docs
- understanding lockfiles and caches

## Two ways to read it

- use the `Core Commands` group if you want operational flows
- use `Advanced Topics` if you need to understand why the system behaves the way it does
