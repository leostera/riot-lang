This page covers `riot run` from the package-management side.

## The important distinction

In the Toolchain tab, `riot run` is about the normal local execution loop.

Here, the important questions are:

- which package is being selected
- which binary is being executed
- whether local or remote package materialization is involved
- how execution fits into wider package workflows

If you are trying to understand execution as part of dependency management rather than editing, start
here. For the everyday loop, read [Toolchain / riot run](/toolchain/riot-run).
