---
title: "riot update"
description: "Refresh dependency selections intentionally instead of drifting implicitly."
---

`riot update` is for the moment when you want dependency movement to be explicit.

## What it should mean

- newer compatible versions can be selected
- lockfile state can move forward intentionally
- the workspace gets a deliberate dependency refresh instead of accidental churn

## When to use it

- before a release
- during dependency hygiene work
- after adding a package and deciding to refresh the graph
- when debugging whether a problem is version-selection-related

The point is control, not constant motion.
