Riot's RFDs are the design record for the stack.

They cover:

- current implemented system snapshots
- accepted or presented future work
- design rationale for the CLI, runtime, registry, package management, and language tooling

This docs site republishes the RFDs from the repository's `docs/rfds/`
directory so they are readable alongside the rest of the stack docs.

## How to read them

If you want the current shape of Riot, start with the implemented RFDs around:

- the build system
- the runtime
- the toolchain
- package management
- the registry and sparse index

If you want the near-term direction of travel, look at the `presented` RFDs.
At the time of writing, those include work like:

- local artifact publishing
- the Riot language server
- the incremental typechecker
- richer actor runtime work

## Current status vocabulary

RFDs use a small lifecycle vocabulary:

- `presented`: written up, but not yet accepted or implemented
- `accepted`: approved, but not fully implemented yet
- `implemented`: reflected in the current repo state
- `rejected`: explicitly not moving forward

The sidebar on this section includes the full set of RFD documents mirrored from
the repository.
