---
title: "Neovim"
description: "Use Riot from Neovim with command-driven workflows and a tight editor-terminal loop."
---

Neovim is a natural fit for Riot if you prefer a command-driven environment and want the editor to stay
close to the terminal workflow instead of abstracting it away.

## Recommended setup

1. open the workspace root
2. install the plugin from `editors/riot.nvim`
3. expose the common Riot commands directly in your normal keymaps or commands

The plugin should make Riot easier to drive from Neovim. It should not invent a second project model.

## The loops worth optimizing

The highest-value bindings are usually:

- `riot check`
- `riot fmt`
- `riot fix`
- `riot build`
- `riot test`

These commands should feel one keystroke away, because they are the commands you will hit most.

## What a good Neovim setup feels like

Good Neovim integration with Riot feels:

- predictable
- package-aware
- terminal-friendly
- lightweight

If the setup starts to feel magical or stateful in a way the CLI does not explain, it is probably too
complicated.
