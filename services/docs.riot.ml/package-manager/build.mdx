This page is about `riot build` as a package-management event, not just as a development loop.

## The important promise

If a package builds, Riot should already have:

- resolved the dependency graph it needs
- materialized dependencies consistently
- kept the lockfile and local dependency state aligned with the workspace rules

That is why `riot build` appears in both the Toolchain and Package Manager tabs. The same command sits
at the boundary between local development and dependency management.

Read [Toolchain / riot build](/toolchain/build) for the day-to-day editing loop.
