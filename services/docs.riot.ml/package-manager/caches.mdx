Riot uses both local and wider cache surfaces because package workflows are too expensive to treat every
step as a full cold start.

## Why caches exist

- to avoid redownloading and rebuilding unnecessarily
- to make repeated local loops fast
- to keep package workflows scalable as the workspace grows

## The two questions users usually have

1. what is cached where?
2. when should I trust the cache versus clean it?

This page is where those answers belong. The operational escape hatch remains [riot clean](/package-manager/clean).
