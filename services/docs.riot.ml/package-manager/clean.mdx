---
title: "riot clean"
description: "Clean package-manager state, caches, and build artifacts when you need a known-good baseline."
---

This is the package-manager view of `riot clean`.

## Use it for

- recovering disk space
- resetting cached dependency state
- forcing a known-clean package-materialization path
- validating that a build or publish flow does not depend on stale local artifacts

## Use it sparingly

If `riot clean` becomes part of your normal loop, the underlying workflow probably needs attention.
Cleaning should be a recovery or validation tool, not a substitute for confidence in the package system.
