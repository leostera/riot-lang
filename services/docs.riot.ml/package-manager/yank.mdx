This page exists to reserve the shape of the package lifecycle even though Riot does not expose a
top-level `riot yank` command today.

## Why keep the placeholder

Once a package ecosystem matures, it needs a story for:

- withdrawing a broken release from normal resolution
- keeping package history legible
- making bad versions less footgun-prone without pretending they never existed

When Riot exposes that workflow, this page should describe it here rather than forcing users to infer
it from registry behavior.
